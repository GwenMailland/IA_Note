## 5. Migration Steps

### Phase 1 — Final Sync (06:00–06:30)
```bash
# Verify replication lag
aws rds describe-db-instances --query 'DBInstances[*].PendingModifiedValues'

# Confirm all services healthy
./scripts/health-check-all.sh

# Enable maintenance mode (read-only API responses)
aws ssm put-parameter --name /scalr/maintenance_mode --value true
```

### Phase 2 — DNS Cutover (06:30–07:00)
```bash
# Update Route53 to point api.scalr.io to new ALB
aws route53 change-resource-record-sets \
  --hosted-zone-id Z1234567890 \
  --change-batch file://dns-cutover.json

# Monitor DNS propagation (target: < 5 min with TTL=60s)
watch -n 10 dig +short api.scalr.io
```

### Phase 3 — Verification (07:00–08:30)
- Run production smoke tests: `./scripts/smoke-test-prod.sh`
- Monitor error rates in CloudWatch (target: < 0.1%)
- Monitor p95 latency (target: < 100ms)
- Manually test critical paths: login, create note, generate document, billing

### Phase 4 — Stabilization (08:30–10:00)
- Disable maintenance mode
- Monitor all dashboards
- Standby for rollback if needed
- First status update to customers at 08:00

## 6. Rollback Procedure

**Decision to rollback**: Error rate > 1% OR p99 > 2s for 5+ minutes

```bash
# Step 1: Revert DNS (instant customer impact reduction)
aws route53 change-resource-record-sets \
  --hosted-zone-id Z1234567890 \
  --change-batch file://dns-rollback.json

# Step 2: Disable feature flags
./scripts/rollback-feature-flags.sh

# Step 3: Verify monolith healthy
curl https://api.scalr.io/health

# Step 4: Notify stakeholders
./scripts/notify-rollback.sh "Rolled back at $(date -u)"
```

**Target rollback time**: < 15 minutes from decision to traffic restored on monolith.
